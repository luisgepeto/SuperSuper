# SVC Multi-Store Training - Plan v3

## Goal

Expand the SVC training dataset from Wegmans-only (24,833 products) to include Walmart and HEB, then retrain and validate whether the multi-store model improves accuracy. The new model is only promoted if it **outperforms the current Wegmans-only model** (82.4% test accuracy).

### Why More Data

The current SVC was trained on a single store's product taxonomy. Different supermarkets categorize products differently (e.g., Wegmans groups coffee creamer under Dairy, Walmart might group it under Beverages). Training on multiple taxonomies teaches the SVC more robust decision boundaries that generalize better to how real users describe products.

### Success Criterion

The multi-store model must achieve **higher test accuracy than 82.4%** on a held-out test set drawn from all stores. If it doesn't, we keep the current Wegmans-only model and investigate why.

## Current State

- **Wegmans**: 24,833 products, 105 leaf categories, fully scraped and mapped
- **SVC model**: Direct Linear SVC, 13 app categories, 82.4% test / 85.5% full accuracy
- **Model in production**: `public/models/svc_model.json` (100.5 KB), cached by service worker
- **Pipeline**: generate_embeddings.py -> evaluate_baseline.py -> evaluate_svc.py -> export_model.py

## Target Supermarkets

| Store | Website | Tech Stack | Pagination | Store Selection | Notes |
|-------|---------|------------|------------|-----------------|-------|
| Walmart | walmart.com | React SPA | `?page=N` (40-50 items/page) | Store ID in session/URL | Filter in-store items ONLY; exclude marketplace sellers |
| HEB | heb.com | SPA + Incapsula bot protection | TBD (likely infinite scroll or pagination) | Store selector | Texas-only; pick HEB Plus! in "wet" county for alcohol |

### Walmart: In-Store Only

Walmart's online catalog includes products available for shipping only, online-only marketplace items, and items that are **not physically present in stores**. Since SuperSuper is an in-store-only shopping experience, the scraper must:
- **Only include products available in-store** at the selected location
- Use in-store availability filters (e.g., `fulfillmentMethod=IN_STORE` or pickup filter) when browsing categories
- Exclude any product that is "Online only", "Shipping only", or only available via third-party marketplace sellers
- The key criterion is: **would a shopper find this product on the shelf at this store?**
- Select a Supercenter in a permissive state (e.g., Texas, New York) that sells beer, wine, and spirits

**robots.txt**: `/browse/` is allowed. `/search/` and `/api/` are disallowed. Pagination via `?page=N` is standard.

### HEB: Texas With Alcohol

HEB operates only in Texas (and Mexico, which we skip). Texas has wet/dry counties that determine alcohol availability. The scraper must:
- Select an HEB Plus! store in a "wet" county (e.g., Austin, Houston, San Antonio)
- HEB Plus! stores have the broadest catalog including beer, wine, and an attached spirits section
- Category URL pattern: `/category/shop/<department>/<category>/<id>`
- Incapsula (Imperva) bot protection requires Playwright with headful or stealth mode

**robots.txt**: `/category/` is allowed (only `/category/*?*` with query params is disallowed). `/search/` and `/graphql` are disallowed.

## Phases

### Phase 1: Walmart Scraper

Build `scripts/scraper/walmart_scraper.py` following the same patterns as the Wegmans scraper.

**Technical approach:**
- Use Playwright (Walmart is a React SPA)
- Navigate to `/browse/food/<category_id>` pages
- Set store to a Walmart Supercenter in a permissive location (sells alcohol, large inventory)
- Paginate using `?page=N` parameter (40-50 items per page)
- Filter to in-store products only: use availability filters or fulfillment badges to exclude online-only and shipping-only items. Only products physically on the shelf at the selected store should be included.
- Extract: product name, full category hierarchy (breadcrumbs), source URL
- Same JSONL output format as Wegmans
- Same features: resume support, streaming batch writes (flush every 30 products), progress tracking

**Category discovery:**
- Start from the Food department page (`/browse/food/976759`)
- Recursively discover subcategories from navigation links
- Build a category index before scraping products (for resume support)

**Store selection:**
- Pick a Walmart Supercenter with: grocery, beer/wine/spirits, large product count
- Document the store ID and location in the script constants

**Output:** `scripts/scraper/data/walmart_YYYYMMDD.jsonl`

### Phase 2: HEB Scraper

Build `scripts/scraper/heb_scraper.py` following the same patterns.

**Technical approach:**
- Use Playwright with headful mode (Incapsula bot protection requires realistic browser behavior)
- Add random delays between requests (2-5 seconds) to avoid triggering bot detection
- Navigate to category pages at `/category/shop/<department>/<id>`
- Set store to an HEB Plus! in a wet Texas county (e.g., Austin area)
- Handle pagination (determine if infinite scroll or page links during development)
- Extract: product name, full category hierarchy, source URL
- Same JSONL output format

**Bot protection mitigation:**
- Run headful (not headless) -- Incapsula fingerprints headless browsers
- Set realistic viewport, user agent, and timezone
- Add human-like delays between page loads
- Limit concurrent requests (sequential only)

**Store selection:**
- Pick an HEB Plus! in Austin, TX (Travis County is "wet" -- full alcohol sales)
- Document the store and confirm alcohol categories are visible

**Output:** `scripts/scraper/data/heb_YYYYMMDD.jsonl`

### Phase 3: Category Mapping

Create category mappings for Walmart and HEB leaf categories to the app's 13 categories.

**Scripts:**
- `scripts/scraper/map_categories.py` -- a utility that:
  1. Reads a JSONL file
  2. Lists all unique leaf categories with product counts
  3. For each unmapped category, suggests the best app category using the current SVC model
  4. Outputs a mapping dict that can be reviewed and corrected manually
  5. Saves the final mapping to a JSON file

**Mapping files:**
- `scripts/scraper/data/mappings/wegmans_mapping.json` (extracted from existing EVAL_MAPPING)
- `scripts/scraper/data/mappings/walmart_mapping.json` (new)
- `scripts/scraper/data/mappings/heb_mapping.json` (new)

Having separate mapping files per store makes the pipeline cleaner -- each store's categories are mapped independently, and the training script reads all mappings.

### Phase 4: Multi-Store Training

Retrain the SVC using combined data from all stores.

**Scripts:**
- Update `scripts/scraper/generate_embeddings.py` to accept multiple JSONL files and generate a combined embedding matrix
- Update `scripts/scraper/evaluate_svc.py` (or create `train_multi_store.py`) to:
  1. Load embeddings from all stores
  2. Apply each store's category mapping to produce app-category labels
  3. Split combined dataset 80/20 (stratified by store + category)
  4. Train Linear SVC with hyperparameter search (C values via 5-fold CV)
  5. Report per-store accuracy breakdown (to identify if one store's data helps or hurts)
  6. Save model only if test accuracy > 82.4%

**Evaluation includes:**
- Overall accuracy vs baseline (82.4%)
- Per-category accuracy comparison (old vs new)
- Per-store accuracy breakdown (does Walmart data help? Does HEB data help?)
- Confusion matrix comparison

### Phase 5: Conditional Model Promotion

Only promote the multi-store model if it outperforms the current one.

**Decision criteria:**
- Overall test accuracy > 82.4%
- No category regresses by more than 5pp (a store might introduce noise for certain categories)
- If accuracy improves: run `export_model.py` to overwrite `public/models/svc_model.json`
- If accuracy doesn't improve: keep current model, document findings, investigate why

**If promoted:**
- Commit new model to `public/models/svc_model.json`
- Update `docs/plans/svc/v3.md` with results
- The browser code (`categoryClassification.js`) doesn't change -- it just loads the new model file

## Scraper Features (Required for All)

Every scraper must include these features (established with the Wegmans scraper):

1. **Resume support**: Save a category index with progress status. `--resume` flag to pick up where we left off.
2. **Streaming batch writes**: Flush every 30 products to disk (not wait for category to finish).
3. **Progress tracking**: Log current category, products found, and estimated progress.
4. **Store selection**: Documented at top of script with constants. Permissive location with alcohol.
5. **Rate limiting**: Respectful delays between requests (minimum 2 seconds).
6. **In-store only**: All scrapers must only include products physically available on the shelf at the selected store. Exclude online-only, shipping-only, and marketplace items.
7. **robots.txt compliance**: Only scrape allowed paths.

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Walmart blocks scraper | Playwright with stealth, rate limiting, headful mode |
| HEB Incapsula blocks scraper | Headful mode, realistic delays, human-like behavior |
| New data hurts accuracy | Phase 5 gate: only promote if accuracy improves |
| Category mapping errors | Phase 3 utility auto-suggests mappings; manual review |
| Store catalog too large | Apply same Wegmans patterns: streaming writes, resume, batch limits |
| Different product name formats | The embedding model handles this -- trained on varied text |

## Notes

- **No code changes to the browser**: The integration (`categoryClassification.js`, `sw.js`) is already complete. Only the model JSON file changes if a better model is found.
- **Additive, not destructive**: If multi-store training doesn't improve accuracy, we lose nothing. The current model stays in production.
- **Per-store accuracy tracking**: We track accuracy broken down by store to understand which data sources help most. This informs whether to add more stores (Kroger, etc.) in the future.
- **Kroger deferred**: v1 planned Kroger as a target. We defer it until we see results from Walmart + HEB. If accuracy plateaus, adding a 4th store may not help.
- **Category mapping is the bottleneck**: The scraping and training are automated. The manual work is reviewing and correcting category mappings (~200-400 categories per store). The `map_categories.py` utility reduces this by auto-suggesting mappings.
